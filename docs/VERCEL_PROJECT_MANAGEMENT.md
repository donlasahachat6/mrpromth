# คู่มือการจัดการโปรเจกต์ใน Vercel

เอกสารนี้จะแนะนำวิธีการตรวจสอบและลบโปรเจกต์ที่ซ้ำซ้อนใน Vercel เพื่อให้คุณจัดการโปรเจกต์ได้อย่างเป็นระเบียบ

## ปัญหา: มีโปรเจกต์ซ้ำซ้อนใน Vercel

เป็นเรื่องปกติที่อาจเกิดโปรเจกต์ซ้ำซ้อนใน Vercel โดยเฉพาะเมื่อมีการ import repository เดียวกันหลายครั้ง หรือมีการ fork แล้ว import ใหม่ ปัญหาคือโปรเจกต์เหล่านี้จะทำงานแยกจากกัน ทำให้การตั้งค่า Environment Variables หรือการติดตาม deployment สับสนได้

## วิธีตรวจสอบและเลือกลบโปรเจกต์

ทำตามขั้นตอนต่อไปนี้เพื่อระบุและลบโปรเจกต์ที่ไม่ต้องการ:

### ขั้นตอนที่ 1: ไปที่ Vercel Dashboard

- ล็อกอินเข้าสู่ [Vercel Dashboard](https://vercel.com/dashboard) ของคุณ

### ขั้นตอนที่ 2: ค้นหาโปรเจกต์ที่เชื่อมกับ Repository ของคุณ

- ในหน้า Dashboard คุณจะเห็นรายการโปรเจกต์ทั้งหมด
- มองหาโปรเจกต์ที่เชื่อมต่อกับ repository `donlasahachat11-lgtm/mrphomth`

![Vercel Dashboard](https://i.imgur.com/YOUR_VERCEL_DASHBOARD.png) <!-- Placeholder for user's dashboard screenshot -->

### ขั้นตอนที่ 3: ระบุโปรเจกต์ที่ถูกต้อง (ที่ควรเก็บไว้)

โปรเจกต์ที่ถูกต้องควรมีลักษณะดังนี้:

- **เชื่อมกับ `main` branch**: ตรวจสอบว่าโปรเจกต์มีการอัปเดตล่าสุดจากการ push code ไปยัง `main` branch
- **มีการตั้งค่า Environment Variables**: เป็นโปรเจกต์ที่คุณได้ตั้งค่า `NEXT_PUBLIC_SUPABASE_URL` และ `NEXT_PUBLIC_SUPABASE_ANON_KEY` ไปแล้ว
- **มี Production Deployment**: เป็นโปรเจกต์ที่มี URL หลัก (Production URL) ที่คุณใช้งานอยู่

ส่วนโปรเจกต์ที่ควรลบมักจะมีลักษณะตรงกันข้าม:

- อาจเชื่อมกับ branch อื่น หรือไม่มีการอัปเดตล่าสุด
- ไม่มีการตั้งค่า Environment Variables
- เป็น deployment ชั่วคราว (Preview Deployment) หรือไม่มี Production URL

### ขั้นตอนที่ 4: การลบโปรเจกต์ที่ไม่ต้องการ

เมื่อคุณระบุโปรเจกต์ที่ต้องการลบได้แล้ว ให้ทำตามขั้นตอนต่อไปนี้:

1.  **คลิกเข้าไปที่โปรเจกต์** ที่คุณต้องการลบ
2.  ไปที่แท็บ **Settings**
3.  เลื่อนลงมาด้านล่างสุดของหน้า จะเจอโซน **Advanced**
4.  คลิกที่ปุ่ม **Delete Project**

![Delete Vercel Project](https://i.imgur.com/YOUR_DELETE_PROJECT_SCREENSHOT.png) <!-- Placeholder for delete project screenshot -->

5.  Vercel จะขอให้คุณ **พิมพ์ชื่อโปรเจกต์เพื่อยืนยัน** การลบ
6.  หลังจากพิมพ์ชื่อและกดยืนยัน โปรเจกต์นั้นจะถูกลบออกจาก Vercel อย่างถาวร

**ข้อควรระวัง:** การลบโปรเจกต์เป็นการกระทำที่ **ไม่สามารถย้อนกลับได้** โปรดตรวจสอบให้แน่ใจว่าคุณได้เลือกโปรเจกต์ที่ถูกต้องก่อนที่จะลบ

## สรุป

การจัดการโปรเจกต์ใน Vercel ให้เป็นระเบียบจะช่วยลดความสับสนและทำให้การพัฒนาแอปพลิเคชันของคุณราบรื่นขึ้น หากคุณมีโปรเจกต์ที่ซ้ำซ้อน ให้ใช้คู่มือนี้เพื่อระบุและลบโปรเจกต์ที่ไม่จำเป็นออกไป
